# Citing lmstudio-bench

If you use this software in academic work, technical reports, or blog posts, please cite it so others can find the tool and the line of work is traceable.

There is no formal DOI yet; use the repository URL and the version you ran (see `package.json` or `npm ls lmstudio-bench`).

## BibTeX

```bibtex
@misc{villarrubia2026lmstudiobench,
  title        = {lmstudio-bench: {CLI} toolkit for benchmarking and tuning llama.cpp inference},
  author       = {James Villarrubia},
  year         = {2026},
  howpublished = {\url{https://github.com/jamesvillarrubia/lm-studio-bench}},
  note         = {Open source (MIT). Substitute the version string with the npm/git release you used.}
}
```

In LaTeX, add `\usepackage{url}` or `hyperref` so `\url{...}` works.

## Plain text (APA-style sketch)

Villarrubia, J. (2026). *lmstudio-bench* (Version 0.1.0) [Computer software]. https://github.com/jamesvillarrubia/lm-studio-bench

Adjust the version and year if you cite a different release.

## Acknowledgments in prose

> We used [lmstudio-bench](https://github.com/jamesvillarrubia/lm-studio-bench) (MIT license) to design and run llama.cpp benchmarks.

That is enough for many venues; use the BibTeX block where a formal reference list is required.
